## The Revolutionary Insight

The iOS SuperBeads Framework was never really about iOS.

What appeared to be an iOS development framework was actually **a universal methodology for structured AI collaboration** that happened to wear iOS clothes. The SwiftUI patterns, xcodebuild verification, and iOS-specific skills were merely one instantiation of something far more powerful.

**The universal truth:**

| What We Thought | What It Actually Is |
|-----------------|---------------------|
| iOS task sizing | **Right-sized cognitive work** (any domain) |
| iOS agents | **Specialized thinking modes** (any task) |
| Code verification | **Observable completion signals** (any output) |
| Development sprints | **Goal-oriented progress** (any objective) |
| Code review | **Human oversight at decision points** (any workflow) |

## The Four Universal Principles

These principles work for **ANY** task, not just code:

### 1. Agentic Architecture (Specialized Thinking Modes)

| Universal Role | Purpose | Code Example | Non-Code Example |
|----------------|---------|--------------|------------------|
| **Strategist** | Plan approach, break down work, make architectural decisions | ios-planner designs app structure | Research planner outlines investigation |
| **Executor** | Do the actual work, implement the plan | ios-implementer writes code | Writer creates content |
| **Specialist** | Provide domain expertise, handle specialized aspects | ios-ui-specialist knows accessibility | Data analyst knows statistics |
| **Critic** | Review output, suggest improvements, ensure quality | ios-reviewer checks patterns | Editor reviews prose |

### 2. Task Discipline (Right-Sized Cognitive Work)

**The 10-15 Minute Rule applies to ANY work:**

| Principle | For Code | For Anything |
|-----------|----------|--------------|
| Atomic units | One function, one component | One section, one analysis, one deliverable |
| Clear inputs | Files to read, APIs to use | Context needed, data required |
| Clear outputs | Files to create/modify | Deliverable to produce |
| Completion signal | Tests pass, builds succeed | Output exists, criteria met |
| Verifiable | Can check programmatically | Can confirm objectively |

### 3. Verification Framework (Observable Completion)

Verification isn't just "tests pass." It's **any observable signal that work is complete:**

| Verification Type | Example | Applies To |
|-------------------|---------|------------|
| **Build/Test** | `xcodebuild`, `pytest` | Code |
| **Output Exists** | File created, content generated | Documents, designs |
| **API Success** | Ticket created, email sent | Tool orchestration |
| **Checklist Complete** | All criteria checked off | Any structured work |
| **Human Confirms** | Supervisor approves | Decision points |

### 4. Session Protocols (Consistent Workflow)

Every session, regardless of domain:

```
SESSION START:
1. Query state (where are we?)
2. Report to supervisor (what's the situation?)
3. Get direction (what should we do?)
4. Execute with verification (do it right)

SESSION END:
1. Summarize progress (what did we accomplish?)
2. Update state (record for next session)
3. Note blockers/learnings (what's important?)
4. Prepare handoff (what does next session need?)
```

## Success Criteria

### Core Engine Success

- [ ] Someone can use the Core alone for ANY task type
- [ ] Universal patterns are clearly documented
- [ ] No domain-specific language in Core docs
- [ ] `superbeads` CLI works without any pack installed

### Pack Success

- [ ] Each pack follows consistent structure
- [ ] Installing a pack accelerates that domain
- [ ] Packs extend Core, don't replace it
- [ ] Clear path to create custom packs

### Overall Success

- [ ] Framework guides ANY Claude Code interaction
- [ ] Users report productivity gains across domains
- [ ] Community contributes additional packs
- [ ] The "iOS framework" becomes a footnote in a larger story

## The Vision Statement

> **Universal SuperBeads is not a framework for building software. It's a framework for working effectively with AI on any complex task. The methodology that made iOS development exceptional now applies to everything.**
