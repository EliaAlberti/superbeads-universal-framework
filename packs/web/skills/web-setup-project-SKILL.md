# Web Setup Project Skill

## Description

You MUST use this skill before setting up any new Next.js project or significantly restructuring an existing one. This applies to creating project structure, configuring build tools, and establishing development workflow.

---

## Purpose

Set up complete, production-ready Next.js projects that:
- Use App Router and modern conventions
- Include proper TypeScript configuration
- Have Tailwind CSS configured
- Are ready for testing
- Include essential tooling

## When to Use

- Creating a new Next.js project
- Restructuring an existing project
- Adding tooling to existing project
- Setting up development environment

## Prerequisites

- Project name and purpose clear
- Node.js 18+ installed
- Package manager preference (npm, pnpm, bun)

### Step 6: Create Utility Functions

```ts
// lib/utils.ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

### Step 8: Create Root Layout

```tsx
// app/layout.tsx
import type { Metadata } from "next";
import { Inter } from "next/font/google";
import "./globals.css";

const inter = Inter({ subsets: ["latin"] });

export const metadata: Metadata = {
  title: {
    default: "My App",
    template: "%s | My App",
  },
  description: "My awesome application",
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body className={inter.className}>{children}</body>
    </html>
  );
}
```

### Step 9: Create verify.sh

```bash
#!/usr/bin/env bash
set -e

echo "=== Next.js Project Verification ==="
```
